# Security Policy

## Reporting a vulnerability

If you believe you have found a security issue, please report it privately.

- Preferred: open a private security advisory on GitHub (if enabled)
- Alternative: email the maintainer

Please include:

- description of the issue and impact
- steps to reproduce
- affected versions/commit

## Supported versions

This is a portfolio project. Only the latest commit on `main` is considered supported.

## Notes on secrets

- Do not commit secrets (JWT keys, database URLs, tokens).
- Use environment variables in production (Railway Variables/Secrets).
